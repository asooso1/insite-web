# /build-fix - 빌드 에러 해결

TypeScript 컴파일 에러와 빌드 실패를 최소 변경으로 체계적으로 해결합니다.

## 사용법

```
/build-fix
/build-fix <특정 에러 메시지>
```

## 실행 절차

1. **감지**: `npm run build` 실행 → 전체 에러 수집
2. **분류**: 파일별 그룹화, 의존성 순서대로 정렬 (근본 문제 먼저)
3. **순차 해결**: 에러별 최소 수정 → 빌드 확인 → 다음 에러
4. **안전 확인**: 새 문제 발생 시 사용자에게 확인 요청

## 일반적인 수정 패턴

| 에러 | 수정 |
|------|------|
| 암시적 `any` | 타입 어노테이션 추가 |
| `undefined` 가능성 | null 체크 또는 옵셔널 체이닝 |
| 모듈 찾기 실패 | import 경로 수정 |
| 타입 불일치 | 타입 캐스팅 또는 타입 수정 |

## 중단 조건

다음 상황에서 작업 중단 후 사용자 확인:
- 수정이 새 에러를 유발할 때
- 동일 에러 3회 반복 시
- 구조적 변경이 필요한 경우
- 패키지 설치가 필요한 경우

## 원칙

- 최소 diff 원칙 (광범위한 리팩토링 금지)
- 아키텍처 변경 금지 (architect 에이전트 역할)
- 에러 하나씩 해결 후 빌드 검증
