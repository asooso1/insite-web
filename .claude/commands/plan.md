# /plan - 구현 계획 수립

구현 전 planner 에이전트를 호출하여 단계별 실행 계획을 수립합니다.

## 사용법

```
/plan <구현할 기능 또는 작업 설명>
```

## 실행 조건

다음 상황에서 반드시 사용:
- 신규 기능 구현
- 여러 파일/컴포넌트 변경이 필요한 작업
- 아키텍처 변경
- 복잡한 리팩토링
- 작업 범위가 불명확한 경우

## 플래너 실행 절차

1. **요구사항 분석** — 구현 범위, 성공 기준, 제약 조건 파악
2. **아키텍처 검토** — 기존 코드 구조 및 영향 컴포넌트 파악
3. **단계 분해** — 파일 경로/함수명 포함한 실행 가능한 단계로 세분화
4. **승인 대기** — 명시적 확인 후에만 코드 작성 시작

## 계획 구조

- 요구사항 요약
- 아키텍처 변경 사항
- 단계별 실행 계획 (Action / Why / Dependencies / Risk)
- 테스트 전략
- 위험 완화 방안

## 중요

planner는 **명시적 승인 전 코드를 작성하지 않습니다.**

승인: "진행해", "yes", "proceed" 등
수정: "수정: [변경사항]"

## 연계 명령어

- `/tdd` — 테스트 주도 구현
- `/build-fix` — 빌드 에러 해결
- `/code-review` — 품질 검토
