# /tdd - 테스트 주도 개발

구현 전 테스트를 먼저 작성하는 TDD 워크플로우를 강제합니다.

## 사용법

```
/tdd <구현할 기능 설명>
```

## 적용 상황

- 신규 기능 구현
- 버그 수정 (재현 테스트 먼저)
- 핵심 비즈니스 로직
- API 클라이언트/React Query 훅

## RED → GREEN → REFACTOR 사이클

```
1. RED:     실패하는 테스트 작성 (예상 동작 기술)
2. 확인:    테스트가 실패하는지 검증
3. GREEN:   테스트를 통과하는 최소 코드 구현
4. 확인:    테스트 통과 확인
5. REFACTOR: 코드 품질 개선 (테스트 유지)
6. REPEAT:  다음 기능으로 반복
```

## 커버리지 기준

- 일반 코드: 80% 이상
- 인증/보안/결제 로직: 100%

## 테스트 작성 원칙

- 구현 세부사항이 아닌 동작(behavior) 테스트
- 테스트 간 독립성 유지
- 외부 서비스(API) mock 처리
- 엣지 케이스: null/undefined, 빈 배열, 에러 응답

## 금지사항

❌ 구현 먼저, 테스트 나중
❌ 테스트 실행 생략
❌ 한 번에 과도한 코드 구현

## 연계 명령어

- `/plan` — 테스트 전략 포함한 계획 수립
- `/build-fix` — 컴파일 에러 해결
- `/verify` — 전체 검증
