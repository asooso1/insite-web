# /refactor-clean - 불필요한 코드 제거

미사용 코드, 불필요한 의존성, 중복 함수를 안전하게 제거합니다.

## 사용법

```
/refactor-clean
/refactor-clean <특정 디렉토리 또는 파일>
```

## 실행 절차

1. **감지**: 미사용 export, 의존성, 타입 탐지
2. **분류**: 안전도별 3단계 분류
3. **삭제**: 테스트 통과 기반으로 보수적 제거
4. **통합**: 중복 함수 병합

## 안전도 분류

| 등급 | 항목 | 처리 |
|------|------|------|
| SAFE | 미사용 유틸리티/헬퍼 | 즉시 삭제 가능 |
| CAREFUL | 컴포넌트/라우트 | 사용 여부 검증 후 삭제 |
| RISKY | 설정 파일/진입점 | 조사 없이 삭제 금지 |

## 삭제 방법론

```
1. 테스트 통과 기준선 확인 (GREEN)
2. 코드 삭제
3. 테스트 재실행
4. 실패 시 즉시 되돌리기
```

## 동적 참조 확인

CAREFUL 항목 삭제 전:
- 동적 import 검색
- 설정 파일의 문자열 참조
- 외부 패키지에서의 사용

## 원칙

- **테스트 후 삭제** (역순 금지)
- **한 번에 하나씩** 변경
- 불확실한 경우 보존 우선
- 리팩토링과 정리를 동시에 하지 않음
- 활발한 개발 중 또는 배포 직전 사용 금지
