## 기술 핵심사항

### v1에서 v2로 마이그레이션 시 주의점

1. **메뉴 시스템**
   - v1: 동적 트리 구조 (Vue.js 재귀 컴포넌트)
   - v2: 정적 설정 또는 동적 로드 필요
   - API: `/api/services/menus` 유지 권장

2. **복잡한 UI**
   - v1: Thymeleaf + Vue + Bootstrap
   - v2: Next.js + Tailwind + shadcn/ui
   - 특히 BEMS/BECM의 3D 시각화, 차트 등 재설계 필요

3. **실시간 기능**
   - v1: 폴링 또는 WebSocket
   - v2: Server-Sent Events (SSE) 또는 WebSocket
   - React Query의 `refetchInterval` 또는 subscriptions 활용

4. **성능 최적화**
   - v1: 페이지 로드 시 초기 렌더링
   - v2: 코드 스플리팅, 지연 로딩 권장
   - BEMS/BECM 같은 복잡한 모듈은 dynamic import 필수

## 자주 묻는 질문 (FAQ)

**Q: 언제 v1을 완전히 폐기할 수 있나요?**
A: Phase 3 완료 후 (대약 12개월). BEMS/BECM은 사용자 의존도가 매우 높으므로 완전한 기능 구현 후 전환 필요.

**Q: Phase 2 중 어떤 모듈부터 시작해야 하나요?**
A: Analysis → Settings → Fieldwork 순서 권장.
- Analysis: 상대적으로 덜 복잡하고 빠르게 가치 제공
- Settings: 시스템 운영에 필수
- Fieldwork: 모바일 우선, 현장 팀 만족도

**Q: BEMS/BECM을 더 빨리 끝낼 수 있나요?**
A: 기술적으로 가능하지만 비권장. 이들 모듈의 복잡도가 높아서 서두르면 품질 저하. 3D 시각화, 실시간 모니터링, 복잡한 분석 등 충분한 시간 필요.

**Q: v1과 v2를 동시에 운영해야 하나요?**
A: Phase 2까지는 필요 (사용자 만족도). Phase 3부터는 점진적 전환 가능.
