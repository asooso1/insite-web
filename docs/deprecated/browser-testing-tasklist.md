# insite-web 브라우저 테스트 비교 태스크 목록

> **목적:** 레거시 FMS (csp-web @ http://121.133.17.22)와 insite-web (Next.js 15) 간 기능 및 UX 비교 검증
>
> **방법론:** 각 모듈별 화면 비교 → 미구현/개선 항목 도출 → 우선순위 기반 구현 → 완료 검증
>
> **테스트 대상:** work-orders, facilities, clients, patrols, reports, analysis, buildings 모듈
>
> **최종 목표:** Phase 6 완료 (75%) → Phase 7 마무리

## 구현 우선순위

### P0 (필수 완료 - Phase 6 마무리)

1. **빌딩 관리 사이드바 메뉴 추가** ⏳미구현
   - 현재: `/buildings` 페이지 구현됨 → 사이드바 미추가
   - 작업: `src/components/layout/sidebar.tsx`에 메뉴 아이템 추가
   - 권한: 역할별 노출 확인 (csp-was `role_menu` 동기화)
   - 예상 비용: 1~2시간

2. **고객사 목록: window.confirm → AlertDialog 교체** ⏳개선
   - 현재: 삭제 시 `window.confirm()` 사용 (규칙 위반)
   - 작업: `src/app/(modules)/clients/_components/delete-button.tsx` 생성
   - 컴포넌트: `AlertDialog` 사용
   - 예상 비용: 30분

### P1 (주요 기능 - Phase 7)

3. **고객사 목록: 상태 탭 필터 추가** ⏳미구현
   - 현재: 필터 없음
   - 레거시: "활성 / 비활성" 탭
   - 작업: `src/lib/api/client.ts`에 상태 탭 추가, 백엔드 쿼리 파라미터 확인
   - 예상 비용: 2~3시간

4. **고객사 목록: 활성 고객사 행 하이라이트** ⏳미구현
   - 현재: 행 스타일 없음
   - 레거시: 활성 고객사 핑크색 배경
   - 작업: DataTable 행 렌더러에 조건부 클래스 추가
   - 예상 비용: 1시간

5. **설비 목록: 설비이력 유무 필터 추가** ⏳미구현
   - 현재: 기본 필터만 지원 (키워드, 페이지)
   - 레거시: "설비이력 유무" 체크박스 (1005건 중 필터링)
   - 작업: FilterBar에 체크박스 타입 추가, 백엔드 쿼리 파라미터 확인
   - 예상 비용: 2~3시간

### P2 (검증 필요)

6. **패트롤 현황: 계획 상태별 탭 필터 동작 검증** ⏳검증필요
   - 현재: 기본 구현됨, 모든 상태 표현 확인 필요
   - 레거시 상태: 유형/계획명/소속팀/계획일시/계획상태
   - 작업: 브라우저 테스트 → 누락된 상태/필터 도출
   - 예상 비용: 1~2시간

7. **사용자 활동 분석: 탭 + 차트 + 테이블 검증** ⏳검증필요
   - 현재: 구현 완료 상태이나 레거시와 동일성 확인 필요
   - 레거시: 분석A/B/C 탭 + 막대 차트 + 순위 테이블
   - 작업: 브라우저 비교 → 누락된 탭/차트 도출
   - 예상 비용: 2~3시간

8. **수시업무: 엑셀 다운로드 동작 검증** ⏳미검증
   - 현재: 기능 구현됨, 실제 테스트 필요
   - 레거시: 필터링된 데이터 엑셀 다운로드
   - 작업: 브라우저 테스트 → 파일명/형식/데이터 검증
   - 예상 비용: 1시간
